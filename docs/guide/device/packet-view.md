# 报文查看

在进行设备调试和故障排查时，查看系统与设备之间交互的原始报文是非常重要的。平台提供了“报文查看”功能，可以实时展示各协议的通讯报文。

## 进入报文查看

1. 在左侧设备树中，选中需要查看报文的设备。
2. 在右侧上方点击“报文”图标（通常像一个消息对话框图标）或选择“报文查看”。
3. 弹出报文查看的窗口，在此可以看到实时的日志或通讯帧信息。

## 不同协议的报文说明

系统会根据设备所配置的协议类型，展示对应的报文格式。

### Modbus 协议报文
对于 Modbus（如 Modbus RTU / Modbus TCP）设备，报文展示主要包含请求帧和响应帧的十六进制数据。
- **时间记录**：记录报文的发送和接收时间。
- **报文方向**：通常会标记发送（TX）和接收（RX）方向。
- **数据结构**：报文通常涵盖了从站地址、功能码、数据起始地址、数据长度及 CRC 校验码等。
- **报文解析**：平台支持对原始十六进制报文进行自动解析，将各个数据段映射为直观的字段含义（如功能码、具体寄存器的一次值）。

> ![](/doc-img/device/packet-view/2.png)

### IEC 104 协议报文
对于 IEC 60870-5-104 设备，报文具有特定的结构规范，主要表现为 APDU 格式。
- **时间记录**：记录报文的发送和接收时间。
- **报文格式**：包含启动字符（固定格式如 68H）、长度、控制域（I帧、S帧、U帧）以及类型标识等。
- **报文交互**：可以通过报文过程快速确认连接建立验证（U帧）、总召唤命令下发及测点数据上送（I帧）、确认信号（S帧）等流程。
- **报文解析**：支持对复杂的 ASDU 数据单元进行解析，直观展示传送原因、公共地址及各个信息对象的值与品质描述词等。

> ![](/doc-img/device/packet-view/3.png)

### DLT645 协议报文
对于 DLT645 协议设备，报文用于展示抄表通信过程中的具体数据。
- **时间记录**：记录报文的发送和接收时间。
- **报文格式**：涵盖起始符（68H）、地址域、控制码、数据域长度、数据域及校验码等，方便定位通讯或解析问题。
- **报文解析**：支持对抄表返回的原始数据块进行深入解析，例如电表实际的电量度数、状态字或异常标识等。

> ![](/doc-img/device/packet-view/4.png)
